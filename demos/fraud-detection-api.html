Sobre o projeto

Visão Geral

Este projeto simula um cenário real de detecção de fraude em fintechs e meios de pagamento. O pipeline vai desde a ingestão de dados CSV até uma API REST em produção, cobrindo treinamento, serialização, serving, re-treino automático com versionamento e CI/CD.

O modelo analisa 6 atributos de cada transação e retorna a probabilidade de fraude em milissegundos, com logging estruturado de cada predição.

Pipeline de Execução

1
Criar ambiente virtual
Isola as dependências do projeto.
python -m venv venv && .\venv\Scripts\Activate
2
Instalar dependências
FastAPI, Scikit-learn, Pandas, Joblib, Pytest.
pip install -r requirements.txt
3
Treinar o modelo
Lê o CSV, treina o Random Forest e salva o .pkl.
python train/train_model.py
4
Rodar os testes
10 testes cobrindo endpoints, validações e o predictor.
pytest tests/ -v
5
Subir a API
FastAPI com hot-reload. Swagger em /docs.
uvicorn app.main:app --reload
6
Deploy com Docker
Container isolado com healthcheck configurado.
docker-compose up --build

Deploy

Docker & CI/CD

A API é totalmente conteinerizada com healthcheck e volume persistente para o modelo. O GitHub Actions roda os testes e valida o build a cada push.

docker-compose.yml
services:
  api:
    build: .
    ports: ["8000:8000"]
    volumes:
      - ./model:/app/model
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8000/health"]
      interval: 30s
GitHub Actions — ci.yml
jobs:
  test:
    steps:
      - run: pip install -r requirements.txt
      - run: python train/train_model.py
      - run: pytest tests/ -v --cov=app
  docker:
    needs: test
    steps:
      - run: docker build -t fraud-detection-api .
